# ATEC线上赛环境测试需求

## 0. 目标概述
在 **IsaacLab** 中搭建一个可最小运行（Minimum Viable）的仿真工程，包含基础场景、机器人与传感器，并提供：
- 机器人状态与传感器数据的 **回传 / 获取接口（API）**
- 可被外部触发的 **单步运行控制接口（step trigger）**
- 完整的 **README**（安装、依赖、使用示例）

---

## 1. 仿真环境搭建（基础场景 + 机器人 + 传感器）

### 1.1 场景要求
- 平地地面（flat ground / plane）
- 可运行的基础光照（默认即可）

### 1.2 机器人要求
- 机器人数量：**1 台**
- 机器人类型：**任意可用机器人**（例如任意四足/人型机器人均可）
- 机器人需可被仿真正常加载，并在仿真中稳定存在（无需完成运动任务）

### 1.3 传感器要求
- 机载 **RGB 摄像头**：**1 个**
  - 安装位置：固定在机器人本体（例如 base_link / torso / head 等）
  - 输出：RGB 图像帧

---

## 2. 状态与观测数据回传（数据内容与接口）

### 2.1 必须回传的数据类型
需要支持获取以下信息（至少提供一次性读取接口；可选提供订阅/缓存机制）：

1. **关节角度** `q`
2. **关节速度** `dq`
3. **IMU 数据**
   - 角速度 `gyro`
   - 线加速度 `acc`
   - 姿态 `orientation`（四元数或欧拉角）
4. **地形点（Terrain points）**
   - 形式：机器人周围高度采样点
5. **RGB 图像**
   - 当前帧图像数据（与摄像头输出一致）

### 2.2 数据格式要求（建议）
- Python 侧返回类型建议：
  - `numpy.ndarray` 
  - 统一在文档中说明 dtype、shape 与坐标系

---

## 3. 数据获取函数 API（必须实现）

### 3.1 必须提供的函数列表（示例命名，可按工程风格调整）
以下函数需在统一的接口类/模块中提供（例如 `sim_api.py` 或 `env_api.py`）：

- `get_joint_pos()`  
  - 返回：当前关节角度 `q`
- `get_joint_vel()`  
  - 返回：当前关节速度 `dq`
- `get_imu()`  
  - 返回：IMU 数据（至少包含 gyro、acc, orientaton）
- `get_terrain_points()`  
  - 返回：地形点集合（点云/高度点等）
- `get_rgb_image()`  
  - 返回：当前帧 RGB 图像数据（本需求明确要求该函数）

### 3.2 返回内容规范（必须在 README 中写清楚）
每个函数在 README 中必须说明：
- 返回的数据结构（ndarray/tensor/dict）
- 数据 shape（例如 `q: (n_dof,)`，RGB: `(H, W, 3)`）

---

## 4. 外部触发的单步运行控制（Single-Step Trigger）

### 4.1 核心行为
仿真器必须支持 **外部指令触发单步执行**：
- 当外部输入触发信号（例如字符串 `"next"`）时：
  - 执行一次仿真步进：`sim.step()`（或 IsaacLab 对应的 step 逻辑）

### 4.2 外部触发接口形式（实现其一或多种）
1. **Python 侧函数触发**（外部模块调用）
   - 例如 `step_once()` 对外暴露，并由外部进程/脚本调用

### 4.3 状态一致性要求
- 每次单步执行后，`get_*()` 系列接口应能读取到 **该步之后** 的最新状态/观测
